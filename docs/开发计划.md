# 《今天也要送到》开发计划

## 1. 当前目标

先做一个可在浏览器直接打开的 MVP 原型，验证以下内容：

- 老人友好的开局问候与状态问卷；
- 根据状态自动生成轻松路线；
- 3D / 伪 3D 社区送报表现；
- 简单移动与投递；
- 报纸分类和轻度记忆；
- 今日总结与本地保存；
- 预留联机入口。

---

## 2. 阶段规划

### 阶段 0：文档与骨架

产出：

- `docs/游戏企划案.md`
- `docs/开发计划.md`
- `index.html`
- `styles.css`
- `src/main.js`

状态：进行中。

当前视觉方向：

- 画面采用“昭和末期～平成初期日本街区氛围”；
- 不做纯 8-bit 像素复刻，而是做柔和、清楚、怀旧的固定视角 3D；
- 保留老人友好大字 UI；
- 场景加入日式住宅、瓦屋顶、樱花、公园、自动贩卖机、电线杆、町内会元素。
- 渲染升级为 Three.js WebGL 3D 大场景：住宅区、公园、河流、小桥、商店、巴士站、鸟居、田地等。
- 交互做减法：取消多问题问卷和出发前记忆题，开始界面只保留一个“三档今日状态”问题，再选择“步行送报 / 骑单车送报”。
- 今日状态会影响本局户数和速度：有点累更短更慢，还不错为标准，精神很好路线稍长。
- 新反馈调整：地图扩大到初版约 10 倍，改为第三人称跟随视角。
- 首页入口改为“步行送报 / 骑单车送报”两种移动方式选择。
- 步行模式显示老人步行动作；单车模式显示车轮、车架、车把和骑行外观。

---

### 阶段 1：单机可玩原型

目标：

- 完成首页、问卷、任务说明、游戏、结算 5 个流程；
- 可移动角色；
- 可投递报纸；
- 能完成 3～5 户送报；
- 用 Canvas 绘制社区、道路、房子、角色和目标提示；
- 支持键盘、鼠标、触屏按钮。

验收：

- 打开 `index.html` 可以进入游戏；
- 完成问卷后进入送报；
- 靠近目标按投递可完成任务；
- 任务完成后进入总结页。

---

### 阶段 2：陪伴感增强

目标：

- 陪伴角色拥有更多台词；
- 根据昨天记录进行问候；
- 邻居拥有更多记忆点；
- 送达后有感谢反馈；
- 可选择“朗读提示”。

验收：

- 第二天进入游戏时能看到昨日相关问候；
- 玩家迷路或答错时反馈温和；
- 总结页突出“感谢”和“陪伴”，不是分数。

---

### 阶段 3：认知玩法扩展

目标：

- 报纸类型更多；
- 找房线索更丰富；
- 加入路线选择；
- 加入昨日回忆问题；
- 每局 1～3 个轻度思考点。

验收：

- 认知任务不打断送报节奏；
- 答错不惩罚；
- 能根据状态调节问题数量。

---

### 阶段 4：联机预留实现

目标：

- 设计玩家 ID；
- 设计好友与家人绑定数据；
- 设计共享任务结构；
- UI 中加入“结伴送报”“家人留言”入口；
- 单机内先模拟异步留言。

验收：

- 即使没有后端，数据结构也能支持后续扩展；
- UI 不需要大改即可接入服务器。

---

### 阶段 5：真实联机

目标：

- WebSocket / WebRTC 多人同步；
- 合作房间；
- 家人端留言；
- 友好对战；
- 云端保存。

验收：

- 两名玩家可进入同一社区；
- 可看到对方位置和进度；
- 合作完成任务后生成共同感谢卡。

---

### 阶段 6：网页发布

目标：

- 使用 GitHub Pages 发布静态网页；
- 仓库 push 后自动部署；
- 用户可直接访问 `https://krysof.github.io/bicycle/` 游玩。

验收：

- GitHub Actions 部署成功；
- 页面可以直接打开；
- 首页、问卷、任务说明、游戏 HUD 正常显示。

## 3. 第一版文件结构

```text
Bicycle/
├─ docs/
│  ├─ 游戏企划案.md
│  └─ 开发计划.md
├─ index.html
├─ styles.css
├─ package.json
└─ src/
   ├─ main.js          # 启动入口，只负责创建 App
   ├─ app.js           # 流程控制
   ├─ data/            # 邻居、问题等静态数据
   ├─ game/            # 难度、移动、投递规则
   ├─ input/           # 键盘/触屏输入
   ├─ render/          # Three.js 3D 场景绘制
   ├─ state/           # 游戏状态、本地保存
   └─ ui/              # 页面和 HUD
```

## 5. 风险与应对

| 风险 | 应对 |
|---|---|
| 老人觉得像考试 | 所有问题包装成日常对话 |
| 操作困难 | 大按钮、宽判定、慢节奏 |
| 3D 眩晕 | 温和第三人称跟随镜头，移动时平滑过渡，不做快速甩镜 |
| 联机复杂 | 先做数据结构和 UI 预留 |
| 家人参与门槛高 | 后续家人端只做留言、查看和鼓励 |

- [x] 玩家与环境细节增强：玩家增加眼睛、眉毛、耳朵、眼镜、衣领、手部等细节；街道增加更多猫、狗、麻雀、鸭子、兔子、蝴蝶和蜻蜓，并加入昆虫飞行动画。

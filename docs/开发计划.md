# 《今天也要送到》开发计划

## 1. 当前目标

先做一个可在浏览器直接打开的 MVP 原型，验证以下内容：

- 老人友好的开局问候与状态问卷；
- 根据状态自动生成轻松路线；
- 3D / 伪 3D 社区送报表现；
- 简单移动与投递；
- 报纸分类和轻度记忆；
- 今日总结与本地保存；
- 预留联机入口。

---

## 2. 阶段规划

### 阶段 0：文档与骨架

产出：

- `docs/游戏企划案.md`
- `docs/开发计划.md`
- `index.html`
- `styles.css`
- `src/main.js`

状态：进行中。

---

### 阶段 1：单机可玩原型

目标：

- 完成首页、问卷、任务说明、游戏、结算 5 个流程；
- 可移动角色；
- 可投递报纸；
- 能完成 3～5 户送报；
- 用 Canvas 绘制社区、道路、房子、角色和目标提示；
- 支持键盘、鼠标、触屏按钮。

验收：

- 打开 `index.html` 可以进入游戏；
- 完成问卷后进入送报；
- 靠近目标按投递可完成任务；
- 任务完成后进入总结页。

---

### 阶段 2：陪伴感增强

目标：

- 陪伴角色拥有更多台词；
- 根据昨天记录进行问候；
- 邻居拥有更多记忆点；
- 送达后有感谢反馈；
- 可选择“朗读提示”。

验收：

- 第二天进入游戏时能看到昨日相关问候；
- 玩家迷路或答错时反馈温和；
- 总结页突出“感谢”和“陪伴”，不是分数。

---

### 阶段 3：认知玩法扩展

目标：

- 报纸类型更多；
- 找房线索更丰富；
- 加入路线选择；
- 加入昨日回忆问题；
- 每局 1～3 个轻度思考点。

验收：

- 认知任务不打断送报节奏；
- 答错不惩罚；
- 能根据状态调节问题数量。

---

### 阶段 4：联机预留实现

目标：

- 设计玩家 ID；
- 设计好友与家人绑定数据；
- 设计共享任务结构；
- UI 中加入“结伴送报”“家人留言”入口；
- 单机内先模拟异步留言。

验收：

- 即使没有后端，数据结构也能支持后续扩展；
- UI 不需要大改即可接入服务器。

---

### 阶段 5：真实联机

目标：

- WebSocket / WebRTC 多人同步；
- 合作房间；
- 家人端留言；
- 友好对战；
- 云端保存。

验收：

- 两名玩家可进入同一社区；
- 可看到对方位置和进度；
- 合作完成任务后生成共同感谢卡。

## 5. 风险与应对

| 风险 | 应对 |
|---|---|
| 老人觉得像考试 | 所有问题包装成日常对话 |
| 操作困难 | 大按钮、宽判定、慢节奏 |
| 3D 眩晕 | 固定俯视等距视角，不做快速镜头旋转 |
| 联机复杂 | 先做数据结构和 UI 预留 |
| 家人参与门槛高 | 后续家人端只做留言、查看和鼓励 |

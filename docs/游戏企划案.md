# 《今天也要送到》游戏企划案

## 1. 项目定位

《今天也要送到》是一款面向老年玩家的 **3D 网页陪伴型送报游戏**。玩家每天先接受温柔问候，根据今日身体、心情和操作状态调整任务难度，然后在温馨社区中步行或骑单车，把不同类型的报纸送到邻居家。

游戏核心不是“考反应”，而是：

- 让老人感觉有人陪伴、有人关心、有人等待；
- 通过送报、找路、记住邻居和报纸分类，进行轻度自然用脑；
- 预留家人陪玩、好友合作、友好对战等联机能力；
- 用短时、低压力、可重复的日常体验，形成稳定陪伴感。

> 注意：本游戏不以医疗产品宣传，不承诺预防或治疗阿兹海默症。设计目标是支持老年人的社交连接、规律参与和适度认知活动。

---

## 2. 目标用户

### 主要用户

- 60 岁以上老人；
- 动作反应变慢，但愿意玩轻松游戏的人；
- 独居、半独居，或需要日常情感陪伴的人；
- 家人希望通过游戏与老人保持轻度互动的家庭。

### 用户需求

- 操作简单，不怕失败；
- 字大、按钮大、提示清楚；
- 每天有人问候；
- 游戏时间短，3～10 分钟即可完成；
- 有一点点思考，但不能像考试；
- 可以和家人、朋友一起玩或互相留言。

---

## 3. 核心体验

### 一句话体验

> 每天打开游戏，都有人问候你；社区里的邻居等着你的报纸；你慢慢走、慢慢想，把今天的事做好。

### 情绪目标

玩家完成一局后应感到：

- 今天有人记得我；
- 我还能完成一件有意义的事情；
- 我认识这个社区的人；
- 我不是一个人在玩；
- 我可以慢慢来，做错也没关系。

## 4. 画面风格与年代依据

### 4.1 60 岁以上日本老人可能熟悉的游戏年代

以 2026 年为当前年份估算，60 岁日本玩家约出生于 1966 年前后：

- 1978 年左右，日本街机文化兴起时约 12 岁；
- 1983 年 7 月 15 日，任天堂 Family Computer / Famicom 在日本发售时约 17 岁；
- 1990 年 Super Famicom 发售时约 24 岁。

因此，如果该人群年轻时接触过电子游戏，较可能熟悉的视觉记忆包括：

- 1970 年代末的日本街机游戏；
- 1980 年代的 Famicom / FC 家用机；
- 1990 年代初的 Super Famicom / SFC 家用机。

不过，本项目不建议直接做成纯 8-bit 像素风。原因是像素风文字小、信息抽象、可读性较弱，不一定适合所有老人。更适合的方向是：

> **昭和末期～平成初期的日本街区怀旧氛围 + 现代清晰大字 UI + 柔和低压力等距 3D 表现。**

### 4.2 美术设计方向

画面关键词：

- 温暖；
- 怀旧；
- 清楚；
- 低压力；
- 社区感；
- 町内会感；
- 像童年或青年时期记忆里的日本街区。

场景元素：

- 低矮日式住宅；
- 瓦屋顶；
- 邮箱；
- 报纸包；
- 樱花树；
- 公园长椅；
- 自动贩卖机；
- 电线杆；
- 町内会提示牌；
- 柔和朝阳和远景小镇。

UI 原则：

- 字体大；
- 按钮大；
- 高对比度；
- 不用闪烁；
- 少量复古装饰即可，不影响阅读；
- 不让玩家觉得是“旧游戏复刻”，而是“熟悉年代的温柔社区”。

### 4.4 当前原型调整

根据“老人操作不要复杂”的原则，当前原型已做减法：

- 首页保留一个非常短的今日状态问题；
- 取消出发前记忆题；
- 今日状态只分三档：有点累 / 还不错 / 精神很好；
- 然后选择骑单车或步行，选择后直接进入送报；
- “有点累”会减少送报户数并放慢速度，“精神很好”会增加一点路线长度；
- 游戏中只突出一个主要动作：靠近发光目标后投递。

画面升级为 WebGL 3D 大场景，但保持固定等距视角，避免快速旋转镜头造成眩晕。场景内容包括住宅区、公园、河流、小桥、商店、巴士站、鸟居、田地、樱花树、自动贩卖机、电线杆等。

后续根据反馈继续调整为：

- 地图扩大到原型初版约 10 倍；
- 视角从固定等距视角改为温和的第三人称跟随视角；
- 首页明确选择“步行送报”或“骑单车送报”；
- 步行时显示老人步行动作、手脚摆动和邮差包；
- 骑单车时显示单车车轮、车架、车把和骑行姿态；
- 单车路线更远，适合展示大场景；步行路线更短，保持安心节奏。

基础世界规则继续补全：

- 房屋、商店、树木、河流、山体、鸟居、田地等应有基础碰撞，不能直接穿过去；
- 河流用护栏和桥表现，桥可以通过，河道不可进入；
- 山体边缘用地形和矮墙表现，避免玩家“穿山”；
- 控制方式改为第三人称驾驶式：上/下为前进和后退，左/右为轻微转向，而不是平面自由平移；
- 自行车必须看起来像真正的二轮车：前后轮、车架、车座、车把、篮筐和后货架要清楚。
- 第三人称移动参考《生化危机5》一类的角色后方视角：前后键负责前进 / 后退，左右键负责向左 / 向右转身，不做左右平移。
- 美术比例要接近生活感：房屋应明显高于人物，门窗、屋顶、邮箱和人物高度要有合理比例，避免“人物像巨人、房子像玩具”。
- 人物本地朝向必须和移动方向一致：身体正面、鼻子 / 面部、双腿和双臂应朝前行进，不能看起来横着走。
- 角色要有男女造型差异，例如颜色、帽子、头发、胡子、裙装 / 裤装等；随机默认名字时同步随机对应造型。
- 步行和骑车都要能看出角色正在拿着报纸，骑车时也要在手边或车前方显示待投递的报纸。
- 默认体验参考 FC《送报童》：玩家默认骑单车，从道路上出发，沿着街道经过一排排房屋。
- 场景比例应更接近参考图：房屋和庭院占画面较大，骑车人相对较小；道路由黑色车道、灰色人行道、白色路缘和标线组成。
- 步行保留为辅助模式，但主体验应优先服务“骑车送报”。
- 住宅排列改为更接近日本居民区：房屋沿道路两侧退让排列，带小院、围墙、门口邮箱，而不是均匀棋盘式散布。
- 目标提示需要老人一眼看清：使用更粗的橙色地面光圈和更高的半透明光柱。
- 建筑不应重复单一样式。街区应包含 20 种以上建筑类型，例如普通住宅、现代住宅、木造老屋、便利店、超市、医院、诊所、药局、邮局、公寓、大楼、银行、交番、学校、图书馆、咖啡店、食堂、面包店、理发店、花店、书店、鱼店、澡堂、停车场等。
- 道路上增加类似导航系统的醒目大箭头，持续指向下一户目标，减少老人迷路的可能。
- 建筑需要有细节：招牌、门窗、遮阳棚、屋顶设备、阳台、盆栽、门牌、空调外机等，让街区更有生活感。
- 住宅细节应继续增加到“能看出生活痕迹”的程度，例如玄关台阶、门框门把、屋檐雨槽、屋顶瓦线、窗框、外机通风格、门灯、电表、花盆、院内踏石、小门和灌木；商业建筑则增加条纹遮阳棚、玻璃门、侧招牌、屋顶风机、货架或停车线。
- 投递不应瞬间完成。按空格键或“投递报纸”按钮后，角色应做出抛报动作，报纸以弧线飞向目标。
- 报纸送达后，房子要有反应，例如轻微弹跳、出现“ありがとう!”标识，并用漫画式大字气泡显示“えいっ!”“ドン!”“ありがとう!”等反馈。

- 为了避免“看得到但靠不过去”，目标房屋采用面向道路的摆放方式；投递判定点不放在房屋中心，而放在路边可通行区域。
- 目标房屋周围会预留空地，普通住宅、树木和景物不会堵住投递点。
- 游戏支持自动多语言：优先读取浏览器语言，自动切换日文、英文、繁体中文、简体中文；也预留 `?lang=ja/en/zhHant/zhHans` 作为测试入口。
- 多语言范围包括首页、HUD、操作提示、总结、路线名、邻居称呼、报纸名称与送达感谢语。
- 语言选择应是游戏画面内的常驻控件，放在右上角，以地球图标 + 下拉框形式显示，老人或家人可以直接切换语言。
- 语言切换不刷新页面，避免老人误以为游戏中断；切换后任务卡、漫画气泡、招牌与总结应即时更新。
- 开始界面提供玩家名字输入框；默认给出随机礼貌称呼，家人可改成老人熟悉的叫法。
- 邻居 / 客户名称全部使用礼貌称呼，例如“先生 / 女士 / さん / Mr. / Ms.”，避免不尊重的年龄标签。
- 道路规划以整齐宽阔的正交主路为主，不使用会误导玩家的窄小斜路；行人和自行车路人沿道路或人行道移动。
- 投递光圈不再是大范围辅助圈，而是略大于房屋 / 院落的明确投递点；这样既清楚，又能要求玩家“骑一段、送一户”。
- 光圈视觉中心必须对准目标房屋中心，实际投递判定仍使用路边可到达点，避免老人需要骑进院子或被围栏卡住。
- 目标房屋使用一个与房屋颜色相近的可见光圈；同一中心另有一个玩家看不见的大触发圈。隐形触发圈总半径为可见光圈半径的 5 倍；进入后角色头顶出现报纸标记，表示此时按空格或按钮即可自动朝房屋投递。
- 导航箭头平时沿道路走；接近终点后直接指向房屋小光圈，避免在终点附近因为道路折线造成箭头来回跳动。
- 道路导航箭头应显示约 10 个，覆盖更远的前方道路，帮助老人提前理解接下来该怎么走。
- 导航箭头应保持横平竖直，符合道路方向；不出现斜箭头，减少方向误读。
- 左右转温情提示必须以玩家当前朝向为准，不能和实际转向相反。
- 若头顶没有报纸标记时按投递，不做失败惩罚，只用漫画气泡温柔提示“再靠近一点点”。
- 自行车需要有可读动画：车轮旋转、脚踏转动、腿部踩踏、身体轻微起伏，避免像静止模型滑动。
- 天空要有怀旧温暖感：渐变蓝天、柔和太阳、慢慢移动的云和远处飞鸟，保持低刺激但不单调。
- 音效与音乐要温和：背景音乐不少于 1 分钟循环，加入低音量自然环境声、鸟鸣 / 花瓣声、踩单车声、投报声和送达提示音，但避免吵杂或高频刺激。
- 进入网站后先出现游戏标题画面，玩家点击“开始”后再进入问候 / 模式选择；这既有正式进入游戏的仪式感，也能解决浏览器需要用户手势才能播放声音的问题。
- 语言选择旁边提供音乐 / 声效独立开关，方便老人或照护者快速调节声音刺激强度。
- 手机端需要提供大号触屏方向按钮和独立投递按钮，HUD 避开底部触控区，防止误触与遮挡。
- 手机端操作改为圆形虚拟摇杆：摇杆上推前进、下拉后退、左右偏移转向；右侧保留独立投递按钮。
- 手机端“今日任务”移动到画面上方，底部只保留虚拟摇杆和投递按钮，避免任务卡与操作区冲突。
- 本地开发使用 Node / Vite 启动，线上仍为静态 GitHub Pages。
- 默认玩家称呼每次进入开始界面都随机生成；如果玩家切换语言，随机称呼应随语言自动本地化，例如中文的“青山女士”切换英文后对应为“Ms. Aoki”。
- 陪伴助手也使用随机名字，且随语言切换本地化，避免固定角色名带来的单调感。
- 背景音乐应更轻快、有散步感：建议使用明亮五声音阶、轻敲击 / 小吉他式节奏、柔和低音与低刺激音量，而不是沉闷长音。
- 背景音乐长度固定为约 1 分钟，并保证循环点没有明显空档或突兀点击声。
- 不同语言使用适合各自文字的字体栈，避免日文、简体、繁体和英文混用同一字体导致显示别扭；场景招牌与 UI 都要同步。
- 自行车转向时要有可见反馈：车把、前轮和车篮随左右转动，车身轻微倾斜，让骑行不再像模型平移。
- 每次开始新关卡时都必须即时随机生成：起点、朝向、目标住户集合、送报顺序和任务户数都不能固定；首个目标应与起点保持一定距离，避免刚开始就在旁边送完。
- 每次开始新关卡时，社区地形和景物布局也要重新生成，包括普通住宅、商店位置、公园、神社、田地、树木、山影和草地色块；道路主网保持清晰，避免老人迷路。
- 随机场景应在标题画面出现前就生成完成，让玩家在第一个标题画面背后看到当天社区；点击开始 / 选择模式时只进入已准备好的地图，不再临时生成造成等待或突变。
- 进入“选择骑车还是步行”的界面时，本局实际使用的随机城市场景就要生成完成，并作为界面背景预览；玩家点击骑车或步行后只进入这张已经准备好的城市，不再临时换地图。路线可以同时预生成，但重点是城市不能在选择后才变化。
- 碰撞体必须和当前随机生成的可见景物同步，不能继续使用旧地图碰撞；院落、屋檐、低矮田地等不应形成看不见的空气墙。
- “今日任务”不应占据画面中央或遮挡道路，应作为左下角轻量任务卡存在；真正的陪伴台词走漫画气泡。
- 导航提示要像真人陪伴，不只是机械箭头：例如“这条路慢慢走就好”“前方差不多向左转”“快到了，看到小光圈就可以投递”。
- 配送途中偶尔出现漫画式“思考框”，例如“嗯……下一个是田中先生呢。”，用于营造陪伴和记忆感；频率要低，不能持续刷屏，也不能挡住投递反馈。
- 社区要有生活流动感：树叶、花瓣、小动物、散步行人和路过自行车可以低频出现。
- 行人数量要明显增加，但仍沿道路 / 人行道移动；行人本身要有可辨识细节，例如头发、帽子、围巾、包、拐杖、购物袋、雨伞、手脚摆动等，让街区更像真实有人生活。
- 当路人或小动物接近时，陪伴角色用温情气泡提醒“前面有人散步，我们慢一点”等，而不是警告式惩罚。
- 第三人称视角下如果建筑遮挡玩家，遮挡建筑临时变半透明，保证老人始终能看到自己在哪里。
- 道路不应形成狭窄通道；主路、人行道和路缘要足够宽，房屋从路边适度后退。
- 中文界面不能混入日文反馈或招牌，除非是刻意保留的日本地名 / 姓名。

## 5. 游戏流程

1. 每日问候  
2. 今日状态问卷  
3. 系统生成今日路线和难度  
4. 家人 / 邻居 / 陪伴角色留言  
5. 领取对应报纸  
6. 步行或骑单车送报  
7. 中途轻度记忆、观察或路线选择  
8. 收到邻居感谢  
9. 今日总结  
10. 可给家人或好友留一句话

## 6. 每日状态问卷

开局不直接进入游戏，而是由陪伴角色询问：

- 今天精神怎么样？
- 今天手指操作方便吗？
- 今天想玩多久？
- 今天想步行还是骑单车？
- 今天想一个人送，还是和朋友一起送？

### 难度映射

| 玩家回答 | 游戏调整 |
|---|---|
| 有点累 | 路线缩短，速度降低，目标减少 |
| 手指不方便 | 投递判定范围变大，自动提示增强 |
| 想玩 3 分钟 | 只送 3 户左右 |
| 状态很好 | 可增加路线长度和报纸分类 |
| 选择步行 | 慢节奏、可随时停 |
| 选择骑车 | 自动减速，仍然宽松判定 |

难度命名避免“简单 / 困难”，改为：

- 轻松路线
- 标准路线
- 活力路线

## 7. 陪伴系统

### 陪伴角色

建议设置一个常驻伙伴，例如小狗、邮局助理或温柔 AI 助手。

功能：

- 每天打招呼；
- 记住昨天的完成情况；
- 中途鼓励；
- 玩家迷路时提示；
- 答错时安慰；
- 完成后总结今天的好事。

示例台词：

> 早上好，今天见到你真高兴。  
> 不着急，我们慢慢走。  
> 你还记得蓝色屋顶是谁家吗？没关系，我们一起看看门牌。  

### 邻居关系

每户邻居有名字、喜好和小故事：

| 邻居 | 特点 | 相关用脑玩法 |
|---|---|---|
| 田中先生 | 喜欢体育版 | 报纸分类 |
| 铃木奶奶 | 门口有红花 | 地标记忆 |
| 山本夫妇 | 蓝色屋顶 | 颜色识别 |
| 小林医生 | 周三要健康报 | 日期规则 |
| 佐藤爷爷 | 喜欢早一点收到 | 路线排序 |

## 8. 适度用脑玩法

用脑内容必须自然融入送报，避免让玩家感觉“被测试”。

### 8.1 报纸分类

不同住户订阅不同报纸：

- 早报
- 体育报
- 健康报
- 园艺报
- 社区通知
- 节日特刊

玩家要把正确报纸送到对应住户。

### 8.2 轻度记忆

示例：

> 你还记得红花门口是哪一家吗？

答错不惩罚，只提示：

> 没关系，我们再看看线索。

### 8.3 路线规划

低难度：系统自动规划。  
中难度：给两个路线让玩家选择。  
高一点：玩家自己决定先送哪户。

### 8.4 观察找房

通过线索寻找目标：

- 红色邮箱；
- 蓝色屋顶；
- 门口有猫；
- 院子有向日葵；
- 公园旁第二户。

## 9. 操作设计

### 基础操作

- 方向键 / WASD：移动；
- 空格 / Enter / 大按钮：投递报纸；
- 鼠标 / 触屏：点击投递；
- 可暂停、可提前结束。

### 老人友好原则

- 字体大；
- 高对比度；
- 不使用复杂组合键；
- 关键按钮始终显示；
- 允许慢慢玩；
- 失败时不扣分、不重来；
- 可选语音朗读。

## 10. 联机模式预留

### 10.1 合作送报

2 名玩家共同完成一条路线。

场景：

- 老人 + 家人；
- 老人 + 朋友；
- 老人 + 护理员；
- 老人 + AI 伙伴。

玩法：

- 一起走完整条街；
- 分头送东街和西街；
- 一人导航、一人投递；
- 最后在公园集合。

### 10.2 异步陪伴

家人不必实时在线，也可以参与：

- 给老人留下今日鼓励语；
- 选择今日轻松任务；
- 查看老人今日是否完成；
- 收到“已完成送报”的提示；
- 老人完成后给家人留一句话。

### 10.3 友好对战

不做强竞技，不强调排名羞辱。

可比较：

- 谁找到目标更快；
- 谁送对报纸更多；
- 谁收到感谢更多；
- 两人一起完成社区目标。

结算语气：

> 今天王爷爷送得很稳，李奶奶找房子很快。两位一起让社区更热闹了！

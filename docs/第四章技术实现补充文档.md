## 4.1 本章目的

本章用于说明本研究所开发的「AI伙伴辅助老年人认知功能训练游戏系统」的具体实现方式。系统以面向老年人的 3D 送报游戏为基础，通过 AI 伙伴、动态难度调整、地图记忆任务、路径规划任务、语音与视觉提示等机制，构成一个兼具陪伴性与认知训练功能的游戏系统。

本系统并非单纯的娱乐游戏，而是将老年人的日常陪伴、空间认知活动、记忆活动和低压力交互结合起来，形成一种可在家庭、养老院或医院康复场景中使用的认知功能支援型游戏系统。

本章重点说明以下内容：

1. 系统整体架构；
2. 游戏执行模块；
3. 行动分析模块；
4. AI 支援判断模块；
5. 音声与视觉 UI 输出模块；
6. 难易度控制机制；
7. 地图与路径规划逻辑；
8. 训练模式与陪伴模式的实现；
9. 手机端与低性能设备优化；
10. 后续实验所需的数据记录设计。

## 4.2 系统整体架构

本系统由四个主要模块构成：

```text
游戏执行模块
    ↓
行动分析模块
    ↓
AI支援判断模块
    ↓
音声・视觉UI输出模块
```

其中，游戏执行模块负责玩家移动、送报任务、地图显示和投递判定；行动分析模块根据玩家的移动、停留、路线偏离、操作失败等行为数据分析当前状态；AI 支援判断模块根据分析结果判断是否需要提示、鼓励、纠错或降低难度；音声与视觉 UI 输出模块则通过语音、漫画气泡、导航箭头、小地图、按钮动画等方式向玩家提供反馈。

该架构的设计目的在于：

- 将游戏操作与认知支援逻辑分离；
- 将玩家行为转化为可分析数据；
- 让 AI 伙伴不是单纯 NPC，而是具有支援判断功能的交互系统；
- 让提示输出不依赖单一方式，而是同时支持语音、文字、图形和动画。

## 4.3 游戏执行模块

### 4.3.1 基本功能

游戏执行模块是系统的基础层，负责实际游戏运行。主要功能包括：

- 玩家角色移动；
- 步行 / 骑自行车模式切换；
- 第三人称视角显示；
- 住宅区地图生成；
- 送报目标设定；
- 报纸投递判定；
- 路人、动物、建筑物、道路等环境要素显示；
- 当前任务进度显示。

游戏采用第三人称视角，玩家从角色后方观察道路和目标。该视角相比俯视视角更具有代入感，同时相比第一人称视角更不容易造成迷路和眩晕，适合老年玩家。

### 4.3.2 送报任务流程

一次送报任务由以下步骤组成：

```text
生成今日任务
↓
选择今日状态与移动方式
↓
确定起点和目标住户
↓
玩家沿道路移动
↓
进入目标投递范围
↓
头顶出现报纸图标
↓
玩家按下投递键
↓
报纸飞向目标房屋
↓
住户出现感谢反馈
↓
进入下一目标
```

系统在每次游戏开始时随机选择若干住户作为今日目标，并根据玩家选择的今日状态调整目标数量、路线长度、提示频率和移动速度。

### 4.3.3 投递判定

为降低老年玩家操作负担，投递判定采用“双层范围”设计。

1. **可见投递圈**：显示在目标房屋附近，用于告诉玩家目标位置；
2. **隐形投递圈**：范围大于可见投递圈，玩家进入后即可投递。

当玩家进入隐形投递圈时，角色头顶显示报纸图标，表示当前可以投递。此时按下空格键或投递按钮，角色会自动朝向目标房屋并投出报纸。该设计避免了需要精确瞄准的问题，降低了操作难度。

如果玩家未进入范围就投递，系统不会判定失败，而是通过 AI 伙伴提示“再靠近一点点”，避免给玩家造成挫败感。

## 4.4 行动分析模块

行动分析模块用于实时分析玩家在游戏中的行为。该模块不直接干预游戏，而是为 AI 支援判断模块提供依据。

### 4.4.1 采集的行为数据

系统主要采集以下行为信息：

| 行为数据 | 含义 | 用途 |
|---|---|---|
| 玩家当前位置 | 当前坐标 | 判断是否偏离路线 |
| 玩家朝向 | 当前面向方向 | 判断导航提示方向 |
| 停留时间 | 是否长时间不动 | 判断是否迷路或犹豫 |
| 路线偏离距离 | 与推荐路线距离 | 判断是否需要路线提示 |
| 投递尝试次数 | 是否多次投递失败 | 判断是否需要操作提示 |
| 与目标距离 | 是否接近目标 | 判断是否显示投递提示 |
| 与行人 / 障碍距离 | 是否存在碰撞风险 | 判断是否提醒减速 |
| 当前任务进度 | 已完成 / 总数 | 判断是否进入总结 |

### 4.4.2 行为状态分类

根据采集数据，系统将玩家状态分为以下几类：

1. **正常移动状态**：玩家沿路线前进，无需额外提示；
2. **接近目标状态**：玩家进入目标附近，需要提示可投递；
3. **停留犹豫状态**：玩家长时间不动，可能不知道下一步；
4. **路线偏离状态**：玩家远离推荐路径，需要导航修正；
5. **重复失败状态**：玩家多次尝试投递但未成功；
6. **交通接近状态**：附近有行人、骑车人或动物，需要安全提示。

该分类使系统能够根据玩家当前情况提供不同类型的支援。

## 4.5 AI 支援判断模块

### 4.5.1 AI 伙伴的定位

本系统中的 AI 伙伴并非单纯的聊天角色，而是承担“状态判断 + 情绪陪伴 + 操作辅助”的复合功能。AI 伙伴根据行动分析模块提供的数据，判断玩家当前是否需要提示，并选择合适的提示方式。

AI 伙伴的主要职责包括：

- 开局问候；
- 今日状态确认；
- 路线导航；
- 迷路提示；
- 投递提示；
- 交通安全提醒；
- 认知训练引导；
- 完成后的鼓励。

### 4.5.2 Yes / No 判断逻辑

佐藤老师指出论文中需要补充 Yes / No 判断逻辑。本系统中的 AI 支援判断可以整理为以下流程：

```text
玩家是否正在游戏中？
    No → 不输出提示
    Yes ↓
玩家是否长时间停留？
    Yes → 输出“慢慢来 / 是否需要找目标”提示
    No ↓
玩家是否偏离推荐路线？
    Yes → 输出方向修正提示
    No ↓
玩家是否接近目标？
    Yes → 显示报纸图标并提示可投递
    No ↓
玩家是否多次投递失败？
    Yes → 输出“再靠近一点点”提示
    No ↓
附近是否有行人或动物？
    Yes → 输出安全提醒
    No ↓
是否到达下一个转弯点？
    Yes → 输出左转 / 右转 / 直行提示
    No → 保持静默
```

该逻辑的重点不是频繁打断玩家，而是在玩家可能需要帮助时提供适度提示。为避免提示过多，系统设置提示间隔，防止短时间内重复输出相同内容。

### 4.5.3 支援强度

AI 支援根据难度和玩家状态分为三档：

| 支援强度 | 触发条件 | 输出内容 |
|---|---|---|
| 低 | 玩家正常前进 | 少量路线提醒 |
| 中 | 玩家犹豫或接近目标 | 导航箭头、漫画气泡、小地图提示 |
| 高 | 玩家偏离路线或重复失败 | 明确方向提示、自动面向目标、自动导航建议 |

在 Easy 模式下，系统更早触发提示；在 Hard 模式下，系统延迟提示，给玩家更多自主判断空间。

## 4.6 音声与视觉 UI 输出模块

### 4.6.1 输出方式

系统通过多种方式向玩家传达信息：

- 语音提示；
- 漫画式对话气泡；
- 导航箭头；
- 小地图；
- 目标光圈；
- 头顶报纸图标；
- 按钮动画；
- 成功投递动画；
- 住户感谢头像。

多通道输出的原因是老年玩家可能存在听觉、视觉或操作差异。仅依靠文字或仅依靠声音都不够稳定，因此系统采用视觉 + 听觉 + 动画反馈的组合方式。

### 4.6.2 漫画气泡

漫画气泡用于表现 AI 伙伴或住户的简短台词。其设计原则为：

- 文字短；
- 自动出现和消失；
- 不强制点击；
- 半透明，避免遮挡地图；
- 配有头像，明确说话者。

例如：

- AI 伙伴：“前面差不多要向左转。”
- 住户：“今天也谢谢你。”

### 4.6.3 小地图与导航箭头

小地图显示玩家位置、目标位置和道路结构。导航箭头贴在地面上，沿道路方向指向目标。接近目标后，箭头直接指向投递圈。

这种设计结合了宏观方向感和局部操作指引：

- 小地图帮助玩家理解整体空间；
- 地面箭头帮助玩家当前行进；
- AI 提示帮助玩家理解下一步动作。

## 4.7 难易度设计

### 4.7.1 难度分类

系统支持 Easy、Normal、Hard 三种难度。为了避免老年玩家产生被评价感，游戏界面可使用更温和的表达，例如“轻松”“标准”“挑战”。

三种难度主要差异如下：

| 项目 | Easy | Normal | Hard |
|---|---|---|---|
| 目标户数 | 少 | 中 | 多 |
| 路线长度 | 短 | 中 | 长 |
| 地图提示 | 多 | 标准 | 少 |
| 小地图信息 | 完整 | 完整 | 部分隐藏 |
| 导航箭头 | 明显 | 标准 | 较少 |
| 投递范围 | 大 | 标准 | 略小 |
| AI 提示 | 频繁 | 适中 | 延迟 |
| 路径规划 | 自动辅助多 | 标准 | 自主判断多 |

### 4.7.2 动态难度调整

除固定难度外，系统还根据玩家表现进行动态调整。

例如：

- 如果玩家长时间停留，增加提示；
- 如果玩家连续偏离路线，显示更明显箭头；
- 如果玩家多次投递失败，扩大提示或自动面向目标；
- 如果玩家顺利完成多个任务，可减少提示频率。

该机制使系统能够适应不同认知水平和操作能力的老年玩家。

### 4.7.3 今日状态与难度

游戏开始时询问玩家今日状态：

- 有点累；
- 还不错；
- 精神很好。

该选择会影响：

- 送报户数；
- 移动速度；
- 路线距离；
- 提示频率；
- 任务复杂度。

这使难度不仅由系统决定，也体现对玩家当天身体和情绪状态的关心。

## 4.8 地图逻辑与空间认知训练

### 4.8.1 地图设计目标

地图设计同时服务两个目标：

1. 让玩家顺利完成送报任务；
2. 通过地图记忆和路径规划刺激空间认知。

地图不是随机迷宫，而是模拟日本住宅区，包括道路、住宅、商店、河堤、学校、公园、神社等地标。

### 4.8.2 地图记忆训练

系统通过以下方式进行地图记忆训练：

- 保留稳定地标；
- 每次路线不同；
- 让玩家记住住户与地标关系；
- 使用路牌帮助形成区域概念；
- 通过 AI 伙伴偶尔回忆“下一位是谁”。

例如，玩家可能逐渐记住：

- 田中先生家在商店通附近；
- 铃木女士家门口有花；
- 医院旁边是小林医生；
- 河堤附近有鸭子和长椅。

### 4.8.3 路径规划训练

在不同难度下，路径规划要求不同：

- Easy：系统给出完整箭头和自动导航；
- Normal：系统显示箭头，但玩家自己操作；
- Hard：减少提示，让玩家根据小地图和地标判断方向。

通过这种方式，玩家可以在不产生强压力的情况下进行空间判断和路线规划。

### 4.8.4 视野范围控制

训练模式中可通过限制信息显示来调整难度：

- Easy：小地图显示全部目标；
- Normal：小地图显示当前目标和道路；
- Hard：只显示玩家周围区域或减少箭头数量。

这种设计可以用于第五章实验中比较不同提示条件对任务完成度和认知负荷的影响。

## 4.9 陪伴模式与训练模式

### 4.9.1 陪伴模式

陪伴模式重点是情绪支持和持续使用。主要功能包括：

- AI 伙伴问候；
- 家人留言；
- 简单送报；
- 明显导航；
- 住户感谢；
- 感谢卡总结。

该模式适合初次使用者、身体状态较差者或希望轻松体验的玩家。

### 4.9.2 训练模式

训练模式重点是认知活动。主要功能包括：

- 地图记忆；
- 路径规划；
- 目标顺序记忆；
- 报纸分类；
- 降低提示频率；
- 错误后给予纠错反馈。

训练模式并不使用考试形式，而是通过送报任务自然引导玩家进行记忆和判断。

### 4.9.3 两种模式比较

| 项目 | 陪伴模式 | 训练模式 |
|---|---|---|
| 目的 | 情绪陪伴、持续使用 | 认知刺激、空间训练 |
| AI 角色 | 主动聊天、鼓励 | 判断提示、纠错反馈 |
| 地图提示 | 明显 | 逐步减少 |
| 任务复杂度 | 低 | 中等 |
| 失败反馈 | 安慰为主 | 安慰 + 纠错 |
| 适用对象 | 初次使用者、低负担需求者 | 可进行轻度训练者 |

## 4.10 手机端与低性能设备优化

由于老年用户可能使用手机、平板或性能较低的电脑，系统进行了多方面优化。

### 4.10.1 操作优化

手机端使用大按钮和虚拟方向控制：

- 左侧控制方向；
- 右侧前进 / 后退；
- 独立投递按钮；
- 任务卡避开触控区域。

### 4.10.2 UI 优化

- 手机端 HUD 缩小；
- 对话气泡半透明；
- 小地图缩小；
- 重要按钮保持足够大；
- 避免多个界面重叠。

### 4.10.3 渲染优化

系统通过以下方式降低负荷：

- 远处建筑降低细节；
- 树木、花草等重复物件批量渲染；
- 控制阴影使用；
- 减少每帧 DOM 更新；
- 降低小地图更新频率；
- 控制移动端像素比。

这些优化保证系统在手机端仍能运行。

## 4.11 数据记录与实验准备

为后续第五章实验，本系统可记录以下数据：

| 数据 | 用途 |
|---|---|
| 完成任务数 | 评价任务完成度 |
| 完成时间 | 评价操作效率 |
| 路线偏离次数 | 评价空间导航能力 |
| 停留次数与时长 | 评价犹豫或迷路情况 |
| 投递失败次数 | 评价操作理解度 |
| AI 提示次数 | 评价支援需求 |
| 自动导航使用次数 | 评价辅助依赖程度 |
| 难度等级 | 区分实验条件 |
| 主观问卷结果 | 评价陪伴感、易用性和负担感 |

实验时应注意个人信息保护，记录数据需匿名化，并在伦理审查中说明数据用途、保存方式和删除方式。

## 4.13 本章总结示例

本章说明了 AI 伙伴辅助老年人认知功能训练游戏系统的具体实现方式。系统以送报游戏为基础，通过游戏执行、行动分析、AI 支援判断、音声与视觉 UI 输出四个模块构成闭环。玩家在游戏中完成地图导航、目标记忆、路径规划和报纸投递等任务，系统根据玩家行为动态判断是否需要提示，并通过 AI 伙伴给予温和的支援。

此外，系统区分陪伴模式与训练模式，使其既可用于日常情感陪伴，也可用于轻度认知训练。难度设计从任务数量、路线长度、提示频率、投递范围和地图信息显示等方面进行调整，以适应不同认知水平和身体状态的老年玩家。通过手机端 UI 优化和性能优化，系统具备在家庭、养老院和医院等场景中进行后续实验的基础。

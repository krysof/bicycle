## 4.1 本章目的

本章用于说明本研究所开发的「AI伙伴辅助老年人认知功能训练游戏系统」的具体技术实现方式。

本系统以邮政 / 报纸配送为题材，通过让玩家在虚拟社区中确认地图、记忆配送目标、规划路线并完成配送任务，使老年人在自然游戏过程中使用记忆力、注意力、判断力、执行功能以及空间认知能力。

本系统并非单纯的自动导航游戏。尤其在训练模式中，系统会让玩家先观察地图和配送目标，然后在提示被部分限制的情况下，根据自己的记忆和判断寻找路线。AI伙伴并不是直接替玩家完成任务，而是在玩家迷路、停留过久、偏离路线、操作失败或遇到障碍时，提供适度的语音、文字和视觉辅助。

本章重点说明以下内容：

- 系统整体结构
- 游戏执行模块
- 行动分析模块
- AI支援判断模块
- 语音与UI输出模块
- 难易度设计
- 地图记忆与路径探索逻辑
- 陪伴模式与训练模式
- 实验所需的行为数据记录设计

## 4.2 系统整体结构

本系统由四个主要模块组成：

```text
游戏执行模块
    ↓
行动分析模块
    ↓
AI支援判断模块
    ↓
语音・UI输出模块
```

游戏执行模块负责玩家移动、地图显示、配送任务、目标判定、投递判定等游戏基础功能。行动分析模块负责分析玩家在游戏中的行为，例如当前位置、停止时间、是否偏离路线、是否接近目标、是否多次操作失败等。AI支援判断模块根据行动分析结果判断是否需要提示、鼓励、纠错或安全提醒。语音・UI输出模块则通过语音提示、漫画式气泡、小地图、导航箭头、目标光圈、按钮动画等形式向玩家反馈。

这种结构的优点是，游戏逻辑、行为分析、AI判断和提示输出相互分离，使系统能够根据玩家状态进行适应性支援。

## 4.3 游戏执行模块

### 4.3.1 基本功能

游戏执行模块是系统的核心，负责玩家实际操作的游戏空间。

主要功能包括：

- 玩家角色移动控制；
- 步行模式与骑车模式切换；
- 第三人称视角控制；
- 配送目标生成；
- 配送顺序管理；
- 报纸 / 邮件投递判定；
- 街区地图、道路、建筑、河流、行人、动物等环境显示；
- 当前任务进度显示；
- 配送完成条件判断；
- 结果画面显示。

本系统采用第三人称视角，使玩家能够从角色后方观察道路、目标和周围环境。与第一人称视角相比，第三人称视角更不容易造成眩晕；与俯视视角相比，第三人称视角更容易产生角色陪伴感和空间方向感，因此更适合老年玩家使用。

### 4.3.2 游戏流程

游戏整体流程如下：

```text
游戏开始
↓
选择今日角色
↓
确认今日身体 / 心情状态
↓
选择步行或骑车
↓
显示地图与配送目标
↓
玩家记忆配送路线
↓
开始配送
↓
玩家根据记忆和提示选择路线
↓
接近目标
↓
投递报纸 / 邮件
↓
住户反馈感谢
↓
前往下一个目标
↓
完成全部配送
↓
显示结果画面
```

该流程对应论文中提出的「地图・配送目标确认」「配送路线记忆」「路径选择」「移动」「结果评价」等步骤。

## 4.4 配送任务设计

### 4.4.1 配送目标信息

每个配送目标包含以下信息：

| 项目 | 内容 |
|---|---|
| 住户姓名 | 例如田中先生、铃木女士等 |
| 配送物类型 | 早报、健康报、社区通知等 |
| 地图位置 | 目标在地图中的坐标 |
| 投递点 | 玩家实际可以投递的位置 |
| 建筑特征 | 屋顶颜色、门口花坛、邮箱、招牌等 |
| 感谢语 | 配送成功后住户说出的反馈 |
| 头像信息 | 漫画式气泡中显示的住户头像 |

配送目标不只是地图上的坐标点，而是具有姓名、房屋特征和反馈内容的“具体人物”。这样设计可以帮助老年玩家将人物、地点和任务内容联系起来，形成自然的记忆活动。

### 4.4.2 投递判定

为了降低老年玩家的操作负担，投递判定采用“双层范围”设计。

第一层是可见目标光圈，用于提示玩家目标位置。第二层是实际投递判定范围，该范围对玩家不可见，但比可见光圈更大。

当玩家进入实际投递范围时，角色头顶会出现报纸图标，表示当前可以投递。此时玩家按下空格键或投递按钮，角色会自动面向目标房屋并投出报纸。

这种设计避免了老年玩家必须精确瞄准的问题，降低了因操作不准而产生的挫败感。

### 4.4.3 成功与未成功反馈

本系统避免使用“失败”“错误”“扣分”等强负面表达。

如果玩家尚未进入投递范围就尝试投递，系统不会播放失败音，也不会立即扣分，而是由AI伙伴温柔提示：

> “再靠近一点点，头上出现报纸标记后就可以投递。”

投递成功时，系统会显示完整反馈：

1. 报纸飞出；
2. 报纸到达目标；
3. 房屋产生反应；
4. 住户头像出现；
5. 住户说出感谢语；
6. 当前配送进度增加。

这样可以使玩家感受到“自己完成了一件有意义的事”，而不仅仅是游戏系统判定成功。

## 4.5 行动分析模块

行动分析模块用于实时分析玩家在游戏中的行为状态，并将分析结果传递给AI支援判断模块。

### 4.5.1 获取的行为信息

系统主要获取以下行为信息：

| 行为信息 | 用途 |
|---|---|
| 玩家当前位置 | 判断是否在推荐路线附近 |
| 玩家当前方向 | 判断左右转提示是否正确 |
| 移动速度 | 判断是否正在移动 |
| 停止时间 | 判断玩家是否迷路或犹豫 |
| 与推荐路线的距离 | 判断是否偏离路线 |
| 与目标的距离 | 判断是否接近配送点 |
| 投递尝试次数 | 判断是否存在操作理解困难 |
| AI提示次数 | 判断玩家对辅助的依赖程度 |
| 自动导航使用次数 | 判断自主探索程度 |
| 与行人 / 动物 / 障碍物的距离 | 判断是否需要安全提醒 |

### 4.5.2 玩家状态分类

根据上述行为信息，系统将玩家状态分为以下几类：

| 状态 | 判定条件 | 支援目的 |
|---|---|---|
| 正常移动 | 玩家沿路线移动 | 不打扰玩家 |
| 停止犹豫 | 一定时间内没有移动 | 判断是否需要提醒或休息 |
| 路线偏离 | 玩家远离推荐路线 | 给出方向修正 |
| 接近目标 | 玩家接近投递点 | 提示可以投递 |
| 投递未成功 | 在范围外投递 | 说明正确操作方式 |
| 交通接近 | 行人、动物靠近 | 提醒减速或暂停 |
| 连续成功 | 多次顺利完成 | 降低提示频率 |

这种状态分类可以让AI伙伴根据不同情况选择不同支援方式。

## 4.6 AI支援判断模块

### 4.6.1 AI伙伴的作用

AI伙伴并不是单纯的聊天角色，而是负责判断玩家当前是否需要帮助的支援系统。

AI伙伴主要承担以下功能：

- 路线引导；
- 操作辅助；
- 投递提示；
- 迷路纠正；
- 停止时的关心；
- 交通安全提醒；
- 鼓励与陪伴；
- 难度调整；
- 认知训练引导。

AI伙伴的设计原则是：不替玩家完成任务，而是在玩家需要帮助时提供适度支援。

### 4.6.2 Yes / No 判断逻辑

AI伙伴的支援判断可以整理为以下 Yes / No 流程：

```text
玩家是否正在游戏中？
├─ 否：不进行支援
└─ 是
    ↓
附近是否有行人、动物或障碍物？
├─ 是：提示减速或暂停
└─ 否
    ↓
玩家是否长时间停止？
├─ 是：询问是否休息，或提示下一目标方向
└─ 否
    ↓
玩家是否偏离推荐路线？
├─ 是：给出方向修正提示
└─ 否
    ↓
玩家是否接近转弯点？
├─ 是：提示左转、右转或直行
└─ 否
    ↓
玩家是否接近目标？
├─ 是：显示可投递标记
└─ 否
    ↓
玩家是否投递未成功？
├─ 是：提示再靠近目标
└─ 否：保持安静
```

该逻辑的重点是避免AI过度干预。只有在玩家可能需要帮助时，AI伙伴才输出提示。

### 4.6.3 支援内容示例

| 玩家状态 | AI支援内容 |
|---|---|
| 接近转弯点 | “前面差不多要向左转。” |
| 接近目标 | “快到了，看到光圈就可以投递。” |
| 长时间停止 | “要不要先休息一下？不着急。” |
| 偏离路线 | “这边有点远了，我们可以往右边那条路看看。” |
| 投递未成功 | “再靠近一点点就可以送到了。” |
| 行人接近 | “前面有人散步，我们慢一点。” |

所有提示都使用鼓励式表达，而不是命令式表达。

## 4.7 难易度设计

### 4.7.1 Easy / Normal / Hard 的差异

本系统设置 Easy、Normal、Hard 三种难度。难度不是单纯改变移动速度，而是同时调整任务数量、路线长度、提示量和地图信息。

| 项目 | Easy | Normal | Hard |
|---|---|---|---|
| 配送目标数量 | 少 | 标准 | 多 |
| 路线距离 | 短 | 标准 | 长 |
| 地图显示 | 常时显示 | 标准显示 | 部分限制 |
| 导航箭头 | 多 | 标准 | 少 |
| AI提示 | 提前提示 | 标准 | 延迟提示 |
| 投递范围 | 大 | 标准 | 略小 |
| 自动导航 | 可使用 | 可使用 | 限制使用 |
| 地图记忆时间 | 长 | 标准 | 短 |

### 4.7.2 今日状态补正

游戏开始时，系统会询问玩家今天的状态：

- 有点累；
- 普通；
- 精神不错。

该选择会影响任务设计：

| 今日状态 | 调整内容 |
|---|---|
| 有点累 | 减少配送目标，降低速度，增加提示 |
| 普通 | 使用标准设定 |
| 精神不错 | 增加目标数量，路线稍长 |

这样可以体现系统对老年玩家身体和心理状态的关注。

### 4.7.3 动态难度调整

游戏过程中，系统也会根据玩家表现动态调整辅助强度。

例如：

```text
玩家长时间停止
→ 增加提示

玩家多次偏离路线
→ 强化导航箭头和AI提示

玩家多次投递未成功
→ 提示投递范围并自动面向目标

玩家连续成功
→ 降低提示频率
```

这种设计可以让系统适应不同认知水平和操作能力的玩家。

## 4.8 地图记忆与路径探索设计

### 4.8.1 基本思想

根据佐藤老师的要求，训练模式中必须包含“老年玩家自己找路”的要素。

因此，本系统不是始终使用完整导航，而是在训练模式中采用“先记忆，后探索，必要时辅助”的结构。

基本流程如下：

```text
显示地图和配送目标
↓
玩家记忆目标位置与顺序
↓
隐藏或减少部分地图信息
↓
玩家根据记忆自行选择路线
↓
如果迷路，AI伙伴给出提示
↓
如果方向错误，AI伙伴进行温和纠正
```

### 4.8.2 地图提示阶段

在配送开始前，系统向玩家展示：

- 当前所在位置；
- 配送目标位置；
- 主要道路；
- 重要地标；
- 推荐配送顺序。

该阶段的目的是让玩家形成空间记忆。

### 4.8.3 路径探索阶段

配送开始后，系统根据难度限制提示信息。

| 难度 | 路径探索条件 |
|---|---|
| Easy | 小地图、箭头、AI提示均显示 |
| Normal | 小地图和标准箭头显示 |
| Hard | 小地图信息减少，箭头减少，AI提示延迟 |

在Hard条件下，玩家需要更多依靠自己对地图的记忆和对地标的判断来寻找路径。

### 4.8.4 纠错反馈

当玩家走错方向时，AI伙伴不应立即给出完整答案，而应分阶段提示。

第一阶段：

> “好像稍微偏离路线了。”

第二阶段：

> “可以看看右边那条路。”

第三阶段：

> “从这边回去会更接近目标。”

这种设计可以保留玩家自主判断的过程，同时避免玩家长时间迷路。

## 4.9 认知功能与游戏要素的对应关系

本游戏通过配送任务自然激活多种认知功能。

| 认知功能 | 游戏要素 | 实现方式 |
|---|---|---|
| 记忆力 | 记住配送目标 | 先查看地图，再根据记忆寻找目标 |
| 注意力 | 确认房屋与目标 | 观察门牌、房屋颜色、光圈 |
| 判断力 | 选择路线 | 判断走哪条路更接近目标 |
| 执行功能 | 按顺序完成配送 | 多个目标按顺序处理 |
| 空间认知 | 理解地图关系 | 根据当前位置和目标位置移动 |
| 持续注意 | 避让路人和动物 | 注意周围环境变化 |

因此，游戏并不是把认知训练做成题目，而是将认知活动融入配送行为中。

## 4.10 陪伴模式与训练模式

### 4.10.1 陪伴模式

陪伴模式的目标是降低不安感，提高持续使用意愿。

特点：

- AI伙伴主动问候；
- 提示较多；
- 小地图和导航箭头常时显示；
- 自动导航可以使用；
- 配送目标较少；
- 更强调住户感谢和家人留言；
- 失败时以鼓励为主。

该模式适合初次使用者、操作不熟悉者或身体状态较差的老年玩家。

### 4.10.2 训练模式

训练模式的目标是激活记忆力、空间认知和判断力。

特点：

- 先显示地图和配送目标；
- 让玩家记忆目标位置与路线；
- 配送开始后限制部分提示；
- 玩家需要自己判断路径；
- AI只在必要时提示；
- 偏离路线时进行分阶段纠错；
- 根据完成情况记录训练数据。

该模式适合已经熟悉操作，或需要进行认知功能训练的玩家。

### 4.10.3 模式比较

| 项目 | 陪伴模式 | 训练模式 |
|---|---|---|
| 主要目的 | 安心陪伴、持续使用 | 认知刺激、空间训练 |
| 地图显示 | 完整显示 | 部分限制 |
| AI提示 | 较多 | 必要时提示 |
| 自主找路 | 较少 | 较多 |
| 错误处理 | 立即安慰和辅助 | 分阶段纠错 |
| 评价重点 | 满意度、安心感 | 完成率、路线判断、记忆表现 |

## 4.11 语音与UI输出

### 4.11.1 输出原则

老年玩家使用时，信息输出必须清楚、温和、低压力。

设计原则如下：

- 字体大；
- 句子短；
- 避免专业术语；
- 使用鼓励式表达；
- 语音和视觉同时提示；
- 对话框半透明，避免遮挡地图；
- 不使用刺耳警告音；
- 不使用“失败”“错误”等负面词。

### 4.11.2 输出形式

| 输出形式 | 作用 |
|---|---|
| 语音提示 | 方向引导、安全提醒、鼓励 |
| 漫画气泡 | 显示AI伙伴或住户的话 |
| 小地图 | 显示当前位置和目标 |
| 地面箭头 | 显示前进方向 |
| 目标光圈 | 显示配送目标 |
| 头顶报纸图标 | 表示可以投递 |
| 按钮动画 | 提醒当前可操作 |
| 住户头像 | 显示感谢反馈 |

这些输出共同构成多通道支援，使听觉或视觉能力存在差异的玩家也能理解游戏状态。

## 4.12 手机端与低性能设备优化

本系统考虑到老年玩家可能使用手机、平板或低性能电脑，因此在交互和画面表现上进行了优化。

### 4.12.1 手机端操作

手机端采用大按钮设计：

- 左侧控制方向；
- 右侧前进按钮；
- 后退按钮较小；
- 投递按钮独立显示；
- HUD避开触控区域。

### 4.12.2 UI优化

- 任务卡缩小；
- 漫画气泡半透明；
- 小地图缩小；
- 按钮保持大尺寸；
- 避免多个UI重叠。

### 4.12.3 性能优化

- 远处建筑降低细节；
- 树木和花草批量渲染；
- 减少实时阴影；
- 降低小地图更新频率；
- 控制手机端渲染负荷。

这些设计可以提升系统在不同设备上的可用性。

## 4.14 本章小结

本章说明了AI伙伴辅助老年人认知功能训练游戏系统的技术实现。系统由游戏执行模块、行动分析模块、AI支援判断模块、语音・UI输出模块组成。游戏执行模块负责配送任务和虚拟环境，行动分析模块负责分析玩家状态，AI支援判断模块根据Yes / No逻辑决定是否介入，语音・UI输出模块则通过多种形式向玩家提供支援。

本系统的重点在于，AI伙伴并非单纯的聊天角色，而是根据玩家行为状态提供适应性支援的系统。同时，训练模式中通过地图记忆、路径探索和分阶段纠错，使玩家能够自己寻找路线，从而自然使用空间认知能力、记忆力、判断力和执行功能。

通过陪伴模式与训练模式的区分，本系统既能提供低压力的日常陪伴体验，也能用于后续认知功能训练实验。

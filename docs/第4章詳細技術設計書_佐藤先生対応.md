## 2. システム概要

本研究で開発するシステムは，高齢者が郵便配達を題材としたゲームを通して，記憶力，注意力，判断力，遂行機能，空間認知能力を自然に活用できるよう設計された認知機能支援ゲームシステムである。

プレイヤーは配達員として街中を移動し，指定された配達先へ郵便物を届ける。ゲーム中には AI パートナーが同行し，プレイヤーの行動状態を分析しながら，必要に応じて音声，漫画風対話表示，ナビゲーション表示，小地図，操作補助などを提供する。

本システムの特徴は，単に目的地まで自動的に案内するのではなく，特に訓練モードにおいて，プレイヤー自身が地図を記憶し，経路を判断しながら配達を進める点にある。AI パートナーは常に答えを提示するのではなく，プレイヤーが迷った場合や誤操作を繰り返した場合に限定して支援する。

## 3. システム全体構成

本システムは，以下の4つの主要モジュールから構成される。

```text
ゲーム実行部
    ↓
行動分析部
    ↓
AI支援判断部
    ↓
音声・UI出力部
```

### 3.1 ゲーム実行部

ゲーム実行部は，プレイヤーが実際に操作するゲーム空間を管理する。主な役割は以下である。

- プレイヤーキャラクターの移動制御
- 歩行モードおよび自転車モードの管理
- 第三人称視点カメラの制御
- 配達先の生成
- 配達順序の管理
- 郵便物の投函判定
- 地図，建物，道路，河川，歩行者，動物などのゲーム環境管理
- 配達完了条件の判定

### 3.2 行動分析部

行動分析部は，プレイヤーの操作およびゲーム内行動を取得し，現在の状態を推定する。

取得対象となる行動情報は以下である。

- 現在位置
- 現在方向
- 移動速度
- 停止時間
- 推奨経路からの逸脱距離
- 目的地までの距離
- 誤投函回数
- 操作入力の有無
- AI支援の利用回数
- 自動ナビゲーションの利用有無
- 歩行者・障害物との接近状態

これらの情報を用いて，プレイヤーが現在「順調に進行しているか」「迷っているか」「目的地に接近しているか」「操作に困っているか」を判定する。

### 3.3 AI支援判断部

AI支援判断部は，行動分析部の結果を基に，AIパートナーがどのような支援を行うべきかを判断する。

支援内容には以下が含まれる。

- 方向案内
- 目的地接近提示
- 誤操作時の説明
- 経路逸脱時の修正
- 停止時の声かけ
- 安全注意
- 励まし
- 難易度調整
- 自動ナビゲーション提案

AIパートナーは，常に支援するのではなく，プレイヤーの自律的な探索を妨げないよう，必要と判断された場合にのみ支援を行う。

### 3.4 音声・UI出力部

音声・UI出力部は，AI支援判断部で決定された支援内容をプレイヤーに提示する。

出力形式は以下である。

- 音声案内
- 漫画風対話吹き出し
- 小地図
- ナビゲーション矢印
- 目的地光圈
- 配達可能マーク
- ボタン強調アニメーション
- 配達成功演出
- 住人の感謝表示

高齢者にとって理解しやすくするため，音声，文字，図形，アニメーションを組み合わせた多重提示を行う。

## 4. ゲーム実行部の詳細設計

### 4.1 基本ゲームフロー

ゲームは以下の流れで進行する。

```text
タイトル画面
↓
プレイヤーキャラクター選択
↓
今日の状態入力
↓
移動手段選択
↓
配達先一覧・地図確認
↓
配達ルート記憶
↓
配達開始
↓
経路探索・移動
↓
配達先確認
↓
郵便物投函
↓
結果フィードバック
↓
次の配達先へ移動
↓
全配達完了
↓
結果画面
```

この流れは，最新版論文の第3章で示されている「地図・配達先確認」「配達ルート記憶」「経路選択」「移動」「結果評価」の流れに対応している。

### 4.2 移動方式

本システムでは，以下の2種類の移動方式を設ける。

| 移動方式 | 目的 | 特徴 |
|---|---|---|
| 歩行モード | 操作負担の低減 | 速度が遅く，短距離ルート向け |
| 自転車モード | 展示性・運動感の向上 | より広い地図を移動可能 |

自転車モードは，今後の動感単車などの外部ハードウェア連携にも拡張しやすい。歩行モードは，身体的負担を抑えたい利用者や，初回体験者向けに使用する。

### 4.3 第三人称視点

カメラはプレイヤー後方から追従する第三人称視点を採用する。

理由は以下である。

- 現在位置と進行方向を把握しやすい
- 第一人称視点より酔いにくい
- 俯瞰視点よりキャラクターへの親近感が高い
- 道路，建物，目的地を同時に確認しやすい

第三人称視点は，老年者に対して過度な没入や視覚負荷を与えず，かつ送報体験を直感的に理解できる視点である。

## 5. 配達タスク設計

### 5.1 配達先の構成

各配達先は，以下の情報を持つ。

| 項目 | 内容 |
|---|---|
| 住人名 | 例：田中さん，鈴木さん |
| 配達物 | 新聞，地域通知，健康新聞など |
| 位置情報 | 地図上の座標 |
| 投函地点 | 道路側の到達可能地点 |
| 建物特徴 | 屋根色，玄関，花壇，郵便受けなど |
| 感謝メッセージ | 配達成功後に表示 |
| 头像情報 | 漫画風表示に使用 |

配達先を単なる座標ではなく，名前，建物特徴，感謝表現を持つ対象として扱うことで，高齢者が人物と場所を結び付けて記憶しやすくする。

### 5.2 投函判定

投函判定は，以下の2段階で構成される。

1. **視覚的投函地点**  
   目的地周辺に表示される小さな光圈。

2. **実際の投函可能範囲**  
   プレイヤーが投函可能と判定される広めの不可視範囲。

プレイヤーが不可視範囲に入ると，頭上に郵便物アイコンが表示される。この状態で投函操作を行うと，キャラクターは自動的に配達先へ向き，郵便物を投げる。

この設計により，高齢者が正確な角度や距離を調整する必要がなくなり，操作負担を軽減できる。

### 5.3 成功・失敗処理

本システムでは，明確な「失敗」表現を避ける。

未到達状態で投函操作を行った場合：

- 減点は行わない
- 失敗音は鳴らさない
- AIパートナーが「もう少し近づきましょう」と案内する

配達成功時：

- 郵便物が飛ぶ
- 建物が反応する
- 住人が表示される
- 住人が感謝を述べる
- 配達数が更新される

高齢者の不安感を軽減するため，失敗よりも次の行動を促すフィードバックを重視する。

## 6. 行動分析部の詳細設計

### 6.1 行動状態の分類

行動分析部では，プレイヤーの状態を以下のように分類する。

| 状態 | 判定条件 | 支援目的 |
|---|---|---|
| 通常移動 | 経路上を移動中 | 支援を控える |
| 停止 | 一定時間移動なし | 迷い・休憩の可能性を判断 |
| 経路逸脱 | 推奨経路から一定距離以上離れる | 方向修正 |
| 目的地接近 | 投函地点に近づく | 投函操作を案内 |
| 投函失敗 | 範囲外で投函操作 | 操作説明 |
| 交通接近 | 歩行者・動物に接近 | 減速・停止案内 |
| 連続成功 | 複数回順調に配達 | 支援頻度低下 |

### 6.2 停止判定

停止判定は，高齢者が操作を迷っている場合と，意図的に休憩している場合の両方を含む。そのため，すぐに指示を出すのではなく，一定時間経過後に柔らかい声かけを行う。

例：

- 「少し休みましょうか」
- 「次の場所を確認しますか」
- 「この先はゆっくり進めば大丈夫です」

### 6.3 経路逸脱判定

経路逸脱は，現在位置と推奨経路の距離によって判定する。

判定例：

```text
現在位置と推奨経路の距離 > 設定閾値
かつ
一定時間その状態が継続
→ 経路逸脱と判定
```

一瞬外れただけでは判定せず，継続時間を考慮することで過剰な介入を防ぐ。

### 6.4 投函失敗判定

投函失敗は，プレイヤーが投函可能範囲外で投函操作を行った場合に発生する。ただし，ゲーム上は失敗として扱わず，操作支援のきっかけとして扱う。

例：

```text
投函操作あり
かつ
投函可能範囲外
→ 操作支援提示
```

## 7. AI支援判断部の詳細設計

### 7.1 支援判断の基本方針

AIパートナーの支援は，以下の原則に基づく。

1. プレイヤーの自律性を妨げない
2. 必要なときだけ介入する
3. 失敗を責めない
4. 次に何をすればよいかを示す
5. 難易度に応じて支援量を変える
6. 訓練モードではヒントを出しすぎない

### 7.2 Yes / No 判断フロー

AI支援判断は以下の Yes / No フローで行う。

```text
[開始]
↓
プレイヤーはゲーム中か？
├─ No：支援なし
└─ Yes
    ↓
    周囲に歩行者・動物・障害物が近いか？
    ├─ Yes：安全提示・減速案内
    └─ No
        ↓
        プレイヤーは長時間停止しているか？
        ├─ Yes：休憩確認または次目標案内
        └─ No
            ↓
            推奨経路から外れているか？
            ├─ Yes：方向修正ヒント
            └─ No
                ↓
                次の曲がり角に近いか？
                ├─ Yes：左折・右折・直進案内
                └─ No
                    ↓
                    目的地に接近しているか？
                    ├─ Yes：投函可能マーク表示
                    └─ No
                        ↓
                        投函操作に失敗したか？
                        ├─ Yes：投函範囲への接近案内
                        └─ No：支援なし
```

この判断フローにより，AIパートナーはプレイヤーの状態に応じて適切な支援を選択する。

### 7.3 支援内容の選択

AI支援判断部は，状態ごとに以下の出力を選択する。

| 状態 | 支援内容 |
|---|---|
| 通常移動 | 基本的に支援なし |
| 曲がり角接近 | 「この先で左へ曲がりましょう」 |
| 目的地接近 | 「そろそろ届けられます」 |
| 停止 | 「少し休みましょうか」 |
| 経路逸脱 | 「こちらの道に戻ると安心です」 |
| 投函失敗 | 「もう少し近づくと届けられます」 |
| 交通接近 | 「前に人がいます，ゆっくり進みましょう」 |

### 7.4 支援頻度制御

高齢者向け支援では，情報量が多すぎると逆に負担となる。そのため，支援にはクールダウン時間を設ける。

例：

- 同じ種類の案内は数秒以内に繰り返さない
- 曲がり角案内は必要な地点でのみ表示
- 思考風メッセージは低頻度にする
- 住人の感謝表示中は次の案内を出さない

## 8. 難易度設計の詳細

### 8.1 難易度パラメータ

Easy，Normal，Hard は以下のパラメータによって制御する。

| パラメータ | Easy | Normal | Hard |
|---|---|---|---|
| 配達先数 | 少ない | 標準 | 多い |
| ルート距離 | 短い | 標準 | 長い |
| 地図表示 | 常時表示 | 常時表示または一部制限 | 一部非表示 |
| ナビ矢印 | 多い | 標準 | 少ない |
| AIヒント | 早めに提示 | 標準 | 遅めに提示 |
| 投函範囲 | 広い | 標準 | やや狭い |
| 自動ナビ | 使用可能 | 使用可能 | 制限または提案のみ |
| 経路記憶時間 | 長い | 標準 | 短い |
| 誤操作補助 | 強い | 標準 | 最小限 |

### 8.2 今日の状態による補正

ゲーム開始時にプレイヤーは今日の状態を選択する。

- 少し疲れた
- 普通
- 元気

この入力により，基本難易度に補正を加える。

例：

| 今日の状態 | 補正内容 |
|---|---|
| 少し疲れた | 配達先数を減らす，速度を下げる，ヒントを増やす |
| 普通 | 標準設定 |
| 元気 | 配達先数を増やす，やや長い経路にする |

この設計により，ゲーム開始時点から「プレイヤーの状態を考慮している」ことを明示でき，AIパートナーによる陪伴感にもつながる。

### 8.3 動的難易度調整

プレイ中の行動に応じて支援量を変更する。

例：

```text
停止時間が長い
→ ヒントを増やす

経路逸脱が多い
→ ナビ矢印を強調する

投函失敗が多い
→ 投函範囲を広く見せる / 自動向き補正

連続成功している
→ ヒント頻度を下げる
```

これは固定的な Easy / Normal / Hard に加えて，実際のプレイヤー状態に応じた適応支援を行うための設計である。

## 9. 地図記憶・経路探索の詳細設計

### 9.1 基本方針

佐藤先生の指摘を踏まえ，本システムでは「高齢者が自分で道を探す」要素を訓練モードの中心に置く。

ただし，すべてを自力で行わせると不安感や失敗感が強くなる可能性があるため，以下の段階的支援を採用する。

```text
地図を見る
↓
配達先を記憶する
↓
地図情報を一部制限する
↓
自分で経路を選択する
↓
迷った場合のみAIがヒントを出す
↓
誤った方向へ進んだ場合は優しく修正する
```

### 9.2 地図提示フェーズ

配達開始前に，プレイヤーへ地図と配達先一覧を提示する。

提示内容：

- 現在位置
- 配達先
- 主要道路
- 地標
- 配達順序

この段階では，プレイヤーが配達先の位置関係を記憶することを目的とする。

### 9.3 記憶フェーズ

プレイヤーは配達先とルートを一定時間確認する。

難易度によって確認時間を変える。

| 難易度 | 地図確認時間 |
|---|---|
| Easy | 長い |
| Normal | 標準 |
| Hard | 短い |

### 9.4 探索フェーズ

配達開始後，難易度に応じて地図や矢印を制限する。

| 難易度 | 探索条件 |
|---|---|
| Easy | 小地図・矢印・AI案内あり |
| Normal | 小地図あり，矢印標準 |
| Hard | 小地図の情報制限，矢印少なめ |

Hard では，プレイヤー自身が地標や記憶をもとに経路を選択する必要が高くなる。

### 9.5 纠错反馈

プレイヤーが誤った方向へ進んだ場合，AI はすぐに答えを出すのではなく，段階的に支援する。

1段階目：軽い気づき

> 「少し道がそれているかもしれません。」

2段階目：方向提示

> 「次は右側の道を確認してみましょう。」

3段階目：明確案内

> 「この道を戻ると目的地に近づきます。」

これにより，プレイヤーの自律的判断を維持しつつ，過度な迷いを防ぐ。

## 10. 認知機能との対応

本ゲームの各要素と認知機能の対応は以下である。

| 認知機能 | ゲーム内要素 | 実装内容 |
|---|---|---|
| 記憶力 | 配達先記憶 | 地図確認後に配達先を覚える |
| 注意力 | 目的地確認 | 建物特徴や住所を確認する |
| 判断力 | 経路選択 | どの道を進むか判断する |
| 遂行機能 | 配達順序計画 | 複数目的地を順に処理する |
| 空間認知 | 地図理解 | 現在地と目的地の位置関係を把握する |
| 持続的注意 | 走行中の確認 | 歩行者，信号，目的地を確認する |

論文では，単に「認知機能を訓練できる」と書くのではなく，どのゲーム要素がどの認知機能に対応しているかを表で示すと説得力が高まる。

## 11. 陪伴モードと訓練モード

### 11.1 陪伴モード

陪伴モードは，継続利用と心理的安心感を重視する。

特徴：

- AIの声かけが多い
- 小地図と矢印を常時表示
- 自動ナビゲーションが利用可能
- 配達先数は少なめ
- 失敗時は励まし中心
- 家族メッセージや感謝カードを重視

適用対象：

- 初回利用者
- 操作に不安がある高齢者
- 軽い日常利用を目的とする場合

### 11.2 訓練モード

訓練モードは，地図記憶と経路判断を重視する。

特徴：

- 地図確認時間を設ける
- 配達開始後に一部情報を制限する
- AIヒントは遅めに提示
- 経路探索をプレイヤー自身に行わせる
- 誤りに対して段階的纠错を行う
- 配達順序や地標記憶を活用する

適用対象：

- 認知機能訓練を目的とする利用者
- 操作に慣れた利用者
- 実験条件として自律探索を評価する場合

### 11.3 両モードの比較

| 項目 | 陪伴モード | 訓練モード |
|---|---|---|
| 主目的 | 安心感，継続利用 | 認知機能刺激 |
| 地図表示 | 常時表示 | 一部制限 |
| AI支援 | 多い | 必要時のみ |
| 自律探索 | 低い | 高い |
| 誤操作対応 | すぐ支援 | 段階的纠错 |
| 評価指標 | 満足度，継続意欲 | 完成率，経路判断，記憶 |

## 12. 音声・UI出力の詳細設計

### 12.1 出力設計の原則

高齢者向けであるため，UI出力は以下の原則に従う。

- 文字は大きくする
- 一文を短くする
- 複雑な専門用語を避ける
- 指示ではなく声かけにする
- 音声と視覚を併用する
- 表示は半透明にして地図を隠さない
- 警告ではなく安心感のある表現にする

### 12.2 音声出力

音声出力は主に以下の場面で使用する。

- ゲーム開始時の挨拶
- 曲がり角案内
- 目的地接近案内
- 誤操作時の説明
- 安全提醒
- 配達成功後の反応
- 結果画面での総括

### 12.3 漫画風UI

漫画風UIは，AIパートナーや住人の発話を視覚的に示すために使用する。

特徴：

- 発話者の头像を表示
- 台詞は短文
- 自動表示・自動消失
- 半透明背景
- PCでは小地図を避ける位置に表示
- スマートフォンでは画面を遮らないサイズに縮小

### 12.4 ナビゲーション矢印

地面に貼り付く形で表示する。

理由：

- 画面上部を遮らない
- プレイヤーや建物と自然に重なる
- 道路標識に近い感覚で理解できる
- 目的地までの方向を直感的に示せる

## 15. 第4章本文への反映例

以下は，論文本文に近い形で使用できる文章例である。

> 本システムでは，プレイヤーの操作状態をリアルタイムに分析し，AIパートナーが必要に応じて支援を行う構成を採用した。AIパートナーは単に会話を行うキャラクターではなく，プレイヤーの停止時間，経路逸脱，目的地接近，誤操作回数などを基に支援の必要性を判断する役割を持つ。これにより，高齢者がゲーム中に迷った場合や操作に困った場合でも，過度な不安を感じることなくゲームを継続できる。

> また，本研究では認知機能訓練の要素として，地図記憶および経路探索を導入した。プレイヤーは配達開始前に地図と配達先を確認し，その位置関係や配達順序を記憶する。その後，難易度に応じて地図表示やナビゲーション情報を制限し，プレイヤー自身が記憶を基に経路を選択する。AIパートナーはプレイヤーが大きく経路を外れた場合や長時間停止した場合にのみヒントを提示するため，自律的な経路探索を維持しながら支援を行うことが可能である。

> 難易度は Easy，Normal，Hard の三段階で構成し，配達先数，経路距離，地図表示，ナビゲーション矢印，AIヒント頻度，投函可能範囲などのパラメータを調整する。Easy では支援を多くし，初めて利用する高齢者でも安心してプレイできるようにする。一方，Hard では地図情報やヒントを制限し，プレイヤーが自ら経路を記憶・判断する必要性を高めることで，空間認知能力や遂行機能の活用を促す。

## 17. まとめ

本システムの技術的特徴は，郵便配達ゲームを基盤としながら，プレイヤーの行動分析，AI支援判断，音声・UI出力を組み合わせることで，高齢者が安心して認知機能訓練に取り組める環境を構築する点にある。

特に，訓練モードでは，地図記憶と経路探索を中心に設計し，プレイヤー自身が道を探す過程を重視する。AIパートナーは完全な自動案内を行うのではなく，プレイヤーの状態を判断し，必要な場面でのみ支援を行う。この構成により，プレイヤーの自律性を保ちながら，迷い，不安，操作困難を軽減することができる。

以上の内容を第4章に追加することで，本研究におけるシステム実装の具体性と，認知機能支援ゲームとしての有効性をより明確に説明できる。
